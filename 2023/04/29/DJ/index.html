在使用 Hexo 搭建的网站中，每个文件目录代表的内容如下：

source目录：该目录是 Hexo 网站的源文件目录，包含了所有博客文章、页面等源代码。其中，_posts目录用于存放博客文章，而pages目录用于存放其他页面的源文件。一些高级用户也可以在该目录中放置其他自定义的脚本或静态资源等。

themes目录：该目录是主题目录，包含了 Hexo 网站所使用的主题模板。用户可以在该目录中切换或自定义主题，以改变网站的外观和风格。

public目录：该目录是 Hexo 网站生成后的输出目录，包含了所有已经生成的静态文件和资源文件。该目录的内容可以直接部署到服务器上进行发布和访问。

node_modules目录：该目录是 Node.js 模块库目录，包含了 Hexo 网站所依赖的所有第三方模块和插件。该目录下的文件一般是不需要手动修改或处理的。

db.json文件：该文件是 Hexo 网站的数据库文件，用于记录网站的配置、分类、标签等元数据信息。

package.json文件：该文件是 Node.js 的项目配置文件，用于记录 Hexo 网站所依赖的所有 Node.js 模块和插件，以及网站的配置信息等。

config.yml文件：该文件是 Hexo 网站的主配置文件，用于记录网站的基本配置，比如网站标题、博客作者、页面风格等。用户可以在该文件中进行自定义配置和修改。
